# 环境变量说明

本文档描述 **`.env.example`** 中的变量含义。实现代码后，以实际读取配置的模块为准；当前为规划与占位。

## 功能开关（预留）

### `PDF_IMPORT_ENABLED`

- **含义**：是否允许走 **PDF 导入** 逻辑（预留）。
- **默认值**：`false`（在 `.env.example` 中已写明）。
- **说明**：PDF 导入为**预留能力**，默认关闭。未实现 PDF 适配器前，即使误设为 `true`，也应由程序在启动或请求时校验并拒绝或降级，并返回明确提示（具体行为在实现阶段写入代码与 README）。
- **建议**：首版开发与联调保持 `false`，仅验证 Word/Markdown 路径。

## 其他变量

`.env.example` 中已用注释列出后续可能用到的变量（LLM、Milvus、MinIO、MongoDB 等），实现时再逐项启用并在此文档补充「必填/可选、示例、安全注意」。

## 文件约定

- 仓库内只提交 **`.env.example`**，勿提交真实 **`.env`**（已在 `.gitignore` 中忽略）。
- 本地开发：复制 `.env.example` → `.env` 后填写密钥与端点。
